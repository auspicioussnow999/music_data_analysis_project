
## 项目简介

本项目旨在通过对音乐数据的收集、处理、分析和可视化，来探索音乐领域的趋势、规律和有价值的见解。项目包括数据获取、数据预处理、数据分析和结果可视化等步骤。

## 目录结构

```plaintext
music_data_analysis_project/
├── README.md
├── data/
│   ├── raw/
│   └── processed/
├── notebooks/
├── src/

│   ├── data_processing.py

```

## 安装依赖

在开始使用本项目之前，请确保已安装以下依赖：

```bash
# 克隆项目仓库
git clone https://github.com/auspicioussnow999/music_data_analysis_project.git

# 进入项目目录
cd music_data_analysis_project

# 创建并激活虚拟环境（推荐使用 venv 或 conda）
python -m venv music_env
source music_env/bin/activate  # Linux/Mac
# 或
music_env\Scripts\activate  # Windows

# 安装项目依赖
pip install -r requirements.txt
```

## 分析报告

### 数据预处理

- **选择最佳聚类数**：从2到10选择最佳聚类数。
- **绘制肘部法则和轮廓系数**：将肘部法则和轮廓系数绘制便于分析。
![k_selection](https://github.com/user-attachments/assets/e3dcf77d-71c9-4f2f-b843-875056bcfb49)


### 数据分析

- **描述性统计**：计算数据的基本统计量（均值、中位数、标准差等）。
- **相关性分析**：分析不同特征之间的相关性。
- **聚类分析**：对音乐流派进行聚类，识别相似的音乐风格。

### 结果可视化

- **PCA降维**：PCA是一种经典的线性降维技术，旨在保留数据的主要方差，将数据集转换为一组线性不相关的变量（主成分），以便在低维空间中展示数据的分布和结构，同时减少数据的维度，加快计算速度，去除噪声和冗余特征。但PCA假设特征之间存在线性关系，可能无法捕捉复杂的非线性结构。
- **t-SNE降维**：t-SNE是一种强大的非线性降维技术，特别适合将高维数据嵌入到二维或三维空间进行可视化。它专注于保留数据的局部结构，能够将具有相似特征的数据点聚集在一起，形成明显的聚类结构，有助于发现数据中的复杂模式和聚类。不过，t-SNE的计算量较大，尤其是对于大型数据集，且结果对超参数敏感，同时它不保留数据的全局结构。
- **聚类重要性条形图**：聚类重要性条形图用于展示各个特征在聚类过程中的重要程度，帮助识别出对聚类结果影响较大的关键特征，从而更好地理解聚类的依据和逻辑，为特征选择提供参考，也可用于解释聚类结果的差异和含义。

## 数据结构概述
`cluster_centers.csv` 文件包含 5 个聚类中心，每个中心对应 12 个特征维度：
- `release_date`（发行日期）
- `length`（长度）
- `popularity`（流行度）
- `danceability`（舞蹈性）
- `acousticness`（声学性）
- `energy`（能量）
- `instrumentalness`（器乐性）
- `liveness`（现场感）
- `loudness`（响度）
- `speechiness`（语音性）
- `tempo`（节奏）
- `time_signature`（节拍签名）

## 聚类中心特征分析

| 聚类中心 | 特征值表现 | 简要说明 |
|----------|------------|----------|
| **聚类中心 1** | `danceability` 和 `acousticness` 较高；`energy` 和 `loudness` 较低 | 可能代表轻松、舒缓的音乐类型 |
| **聚类中心 2** | `instrumentalness`、`liveness` 和 `speechiness` 较高 | 可能代表现场表演或语音元素较多的音乐 |
| **聚类中心 3** | `popularity` 和 `danceability` 较低；`acousticness` 和 `energy` 较低 | 可能代表较为冷门或小众的音乐类型 |
| **聚类中心 4** | `length` 和 `instrumentalness` 较高；`danceability` 和 `energy` 较低 | 可能代表长篇的器乐曲目，如古典音乐或环境音乐 |
| **聚类中心 5** | `tempo` 和 `time_signature` 较高；`acousticness` 和 `energy` 较低 | 可能代表节奏感强但能量较低的音乐类型 |

## 聚类结果解读

- **音乐类型分类**：聚类中心可用于将音乐数据划分为不同类别，每个类别代表一种音乐类型或风格。
- **音乐特征分析**：通过分析每个聚类中心的特征，了解不同音乐类型的主要特征和特性。
- **音乐推荐**：利用聚类结果为用户推荐与其偏好相似的音乐类型。

## 进一步分析建议

- **可视化**：使用降维技术（如 PCA 或 t-SNE）对聚类结果进行可视化。
- **特征重要性**：分析每个特征在聚类中的重要性。
- **聚类效果评估**：使用轮廓系数（Silhouette Score）等指标评估聚类效果。
